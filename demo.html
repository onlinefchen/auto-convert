详细的使用步骤和功能介绍

快速开始

1准备订阅链接

获取你的代理服务商提供的订阅链接，格式通常如下：

https://sub.example.com/link/xxx
提示：订阅链接是代理服务商提供的，包含了所有可用的代理节点信息。

2选择转换方式

🌐 方式一：在线工具（推荐）

直接在浏览器中使用，无需下载任何软件：

打开在线工具

💻 方式二：命令行工具

适合开发者和高级用户：

# 克隆项目 git clone https://github.com/onlinefchen/auto-convert.git cd auto-convert # 一键运行 ./convert.sh https://your-subscription-url

3配置 GitHub Token（可选）

如果需要生成二维码，需要配置 GitHub Token：

  1. 访问：GitHub Settings
  2. 点击 "Generate new token" → "Generate new token (classic)"
  3. 配置 Token：
    • Note: Proxy Config Gist Upload
    • Expiration: 30 days
    • Scopes: ✅ 只勾选 gist
  4. 生成并复制 Token（格式：ghp_xxxxxxxxxxxx）
注意：Token 只显示一次，请妥善保存！

4生成配置文件

在在线工具中：

  1. 输入订阅链接
  2. 选择输出格式（Surge、Clash 或两者）
  3. 输入 GitHub Token（如果需要二维码）
  4. 点击"开始转换"
隐私保护：所有处理都在客户端完成，我们不会保存您的数据。

5导入配置

📱 方式一：扫描二维码

如果配置了 GitHub Token，会生成二维码：

  1. 用手机扫描二维码
  2. 在 Surge/Clash 中添加配置订阅
  3. 使用扫描的链接作为订阅地址

📋 方式二：复制链接

复制生成的 Gist 直链，在客户端中手动添加订阅。

📄 方式三：下载文件

直接复制配置文件内容，保存为 .conf 或 .yaml 文件导入。

常见问题

Q: 为什么需要 GitHub Token？

A: GitHub Token 用于创建私有 Gist 来存储配置文件，这样可以生成二维码供扫描导入。

Q: 我的订阅信息会泄露吗？

A: 不会。私有 Gist 只有知道链接的人才能访问，所有处理都在客户端完成。

Q: 支持哪些代理协议？

A: 目前支持 VMess、Shadowsocks (SS)、Trojan 协议。

Q: 生成的配置文件如何更新？

A: 当订阅内容变化时，重新运行转换工具即可。
